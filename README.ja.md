# vibesubin

**バイブコーダー** のためのケアリングなスキルパック — AI を使って本番のリポジトリを運用しているが、開発者としての訓練を受けていない人向け。

> [English](./README.md) · [한국어](./README.ko.md) · [中文](./README.zh.md)

> **正本は英語版です**。この日本語版は概要翻訳です。詳細なトリガーシナリオ、方法論、トラブルシューティング、コントリビューションガイドは [English README](./README.md) を参照してください。全文翻訳は英語版が安定した後に行います。

## このパックが解決する問題

非開発者が AI にリファクタリング、セキュリティチェック、デプロイを頼んだときに起こる典型的な失敗は、**静かに** 起こります:

- AI が「完了」と言ったのに検証されていない — クラスを移動したが、3 つの呼び出し元がまだ古いパスを参照している
- 3000 行のドキュメントを書き直す過程で、12 個の重要な事実が静かに消える
- セキュリティ「監査」が 30 件の警告を出すが全部誤検知で、本当の脆弱性は見逃される
- ハードコードされた `C:\Users\alice\...` パスがチームメイトの PC からそのままコミットされる
- 「親切な」リファクタリングが存在しない import を追加してしまい、誰も気づかない
- リポジトリが腐り始める: デッドコード、巨大ファイル、高ホットスポット、でも誰も追跡していない
- CI/CD は手動 SSH、ロールバック計画なし、操作者は `workflow_run` という言葉を聞いたこともない
- コミットメッセージが "update" だけ、次の AI セッションは diff を最初から読み直さないと何が変わったか分からない

**これはあなたのせいではありません。** ツールは誰も教えてくれなかった知識を前提にしています。このパックは、その欠けたピースの一部を、AI があなたの代わりに使えるスキルとして提供する試みです。

## 6 つのスキル

それぞれのスキルは自己完結したディレクトリで、[`plugins/vibesubin/skills/`](./plugins/vibesubin/skills/) にあります。名前を覚える必要はありません — AI があなたの質問に応じて適切なスキルを選びます。

| スキル | 役割 |
|---|---|
| **`refactor-safely`** | 任意のリファクタリングを Mikado 方式の依存ツリーとして計画し、葉から上へ実行します。**6 ステップの再帰的検証**（シンボル集合 diff、AST ノード diff、ビルド/テスト、呼び出し側クロージャチェック）で 1:1 の意味的等価性を証明します。トリガー: "リファクタリングして"、"これまだ動く?"、"このファイル分割して" |
| **`audit-security`** | 意図的に小さく、手作業で選ばれたセキュリティスイープ。ハードコードされた秘密、SQL/shell インジェクション、XSS、パストラバーサル、危険なデシリアライゼーション、Cookie フラグ欠如、ワイルドカード CORS、追跡中の秘密ファイルを探します。すべての発見を "真" / "誤検知" / "要確認" に分類。トリガー: "安全?"、"脆弱性ある?"、"監査" |
| **`fight-repo-rot`** | **変更頻度 × 複雑さ** によるホットスポット分析。最もリファクタリング優先度の高いファイルを洗い出します。巨大ファイル、デッドコード、ハードコードされたパス、古い TODO、偏ったインポートグラフも検出します。トリガー: "これ掃除して"、"何が腐ってる"、"デッドコードある?" |
| **`write-for-ai`** | **次の AI セッション** が冷たい状態からプロジェクトを理解できるように最適化されたドキュメント・コミット・PR ライティング。README、`CLAUDE.md`/`AGENTS.md`、conventional commit、PR 本文のテンプレート付き。原則: 散文よりテーブルとチェックリスト、あいまいな参照より絶対パスをバッククォートで、インバリアントを専用セクションに |
| **`setup-ci`** | CI/CD を非開発者向けに基礎から説明し、その後動く GitHub Actions `test.yml` + `deploy.yml` を生成します。concurrency グループ、タイムアウト、秘密情報のクリーンアップ、デプロイ後のヘルスチェックを内蔵。**必須 / 推奨 / オプション / あると便利 / 高コスト** の 5 段階ツリーで構成要素を選ばせます |
| **`manage-config-env`** | 構造的な決定に対する、意見の強いデフォルト: `.env` レイアウト、`.gitignore` テンプレート、依存関係のピン止め、ブランチ戦略（デフォルトは GitHub Flow）、ディレクトリ構造と命名規則、パスポータビリティ監査 |

さらに、傘スキル **`vibesubin`** があります。どこから始めていいか分からないときは `/vibesubin` と打つとこのスキルが起動し、短い質問を 1 つして、適切な専門スキルにルーティングします。

## パック全体を貫く 3 つの価値

すべての設計判断はこの 3 つを最適化しています。衝突時はこの順序で取捨選択します:

1. **保守性** — リポジトリは 6 ヶ月後でも **初見の読者**（多くは新しい AI セッション、もしくは 6 ヶ月後のあなた）に読めるべき
2. **セキュリティ** — 敵対者がすべての diff を読むと仮定して書く
3. **生産性** — 1 つのコマンドで退屈な作業を 1 つ終えるべき。操作者の注意力は最も希少なリソース

## インストール

### Claude Code プラグインマーケットプレイス経由（推奨）

```
/plugin marketplace add subinium/vibesubin
/plugin install vibesubin@vibesubin
/plugin marketplace update    # 後日、更新を取得するとき
```

**自動更新** が得られる唯一のパスです。

### 手動インストール（フォールバック）

```bash
git clone https://github.com/subinium/vibesubin.git
cd vibesubin
bash install.sh         # 各スキルを ~/.claude/skills/ にシンボリックリンクします
```

アンインストール: `bash uninstall.sh`。

### ポータビリティについて

このパックは現在 Claude Code のスキル形式で書かれていますが、**構造自体は移植可能** です。各スキルは自己完結したディレクトリ（Markdown + スクリプト + テンプレート）で、他の skill marketplace や agent フレームワークへの移植コストは低く抑えられています。他プラットフォームのサポートが追加されたら、ここにインストール手順を記載します。

## 日常の使い方

インストール後、ほとんどの場合スキルを名前で呼び出す必要はありません。自然な言葉で頼めば、AI が適切なスキルを選びます:

| あなたが言うこと | トリガーされるスキル |
|---|---|
| "ここを整理して、ただし壊さないで" | `refactor-safely` |
| "さっき変更したけど、まだ動いてる?" | `refactor-safely` |
| "何か漏れてない? 脆弱性ある?" | `audit-security` |
| "うちのリポジトリは大丈夫? 何から直すべき?" | `fight-repo-rot` |
| "README 書いて" / "コミットメッセージ" / "PR" | `write-for-ai` |
| "自動デプロイ設定して" | `setup-ci` |
| "この秘密情報はどこに置くべき?" / "main と dev どっち?" | `manage-config-env` |

迷ったときは `/vibesubin` と打てば傘スキルがガイドします。

## Issue のみ、PR は受け付けていません

本パックはオープンソースですが、**現在 Pull Request は受け付けていません**。以下のような場合は [issue を立てて](https://github.com/subinium/vibesubin/issues) ください:

- 実際のタスクでスキルが失敗した
- あなたが使っている言語 / ランタイムがカバーされていない
- リファクタリングまたはセキュリティパターンが漏れていた
- ドキュメントが非開発者にとって不明瞭だった

メンテナーが issue を確認して直接反映します。パックのトーンの一貫性を保つためです。
